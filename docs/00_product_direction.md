# Photo Studio OS / Product Direction v1

## 定版判断

Photo Studio OS 当前先聚焦一个核心页面：

```txt
Command Center / Brooklyn Studio A visual baseline
```

这不是普通后台首页，而是摄影工作室的主控台：项目推进、SKU 覆盖、QC 健康、风险、审批、AI 检查、执行状态，都应该在一张宽屏界面里被冷静地呈现。

## 当前视觉母版

已锁定方向：

- 近黑背景
- 冷白文字
- 少量橙红风险色
- 三仪表结构
- 中央大仪表作为 Studio Readiness / Project Progress
- 左侧小仪表作为 SKU Coverage
- 右侧小仪表作为 QC Health
- 右侧栏为 Risk Pulse + Approval Queue
- 下方三卡片为 Project Execution / Activity Timeline / AI Inspection Feed

## 设计原则

1. 不做泛 SaaS 后台。
2. 不做游戏 HUD。
3. 不做过亮、过蓝、过科幻的界面。
4. 中央仪表必须是视觉心脏。
5. 右侧风险区只负责提醒，不抢主视觉。
6. 下方卡片负责真实业务感。
7. Canva 用于视觉规范和展示，GitHub + 前端代码是主战场。
